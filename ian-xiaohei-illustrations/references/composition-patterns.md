# Composition Patterns and Originality Rules

## Basic Structure Types

Pick one structure — don't mix too many together.

### Workflow

Good for: input -> processing -> output, content production, AI workflows, automation pipelines.

How to draw it: input on the left, Xiaohei or a strange machine processing in the middle, output on the right, with an orange arrow expressing the main flow direction.

### Partial System

Good for: information sources, filters, databases, renderers, parts of an agent system.

How to draw it: draw only 3-5 core modules, with Xiaohei taking part in one key action.

### Before/After Contrast

Good for: chaos/order, manual/automatic, scattered/gathered, anxious/stable.

How to draw it: chaos on the left, stability on the right, with an orange arrow in the middle. The character can be more exaggerated.

### Character States

Good for: user pain points, the creator's state, too many tools, information anxiety, going from stuck to running smoothly.

How to draw it: 2-4 small states, each with a short caption.

### Conceptual Metaphor

Good for: content factory, information warehouse, black box in the brain, workflow machine, automated daily report.

How to draw it: one large strange object or machine, a small amount of input, one output. It needs to be memorable.

### Layered Method

Good for: methodology frameworks, system hierarchies, capability stacks, layered content systems.

How to draw it: stacked boxes layer by layer — avoid a formal pyramid; have Xiaohei moving bricks or building alongside it.

### Map Route

Good for: from idea to launch, user journeys, content delivery paths, learning paths.

How to draw it: one winding path, a small number of nodes, Xiaohei leading the way or walking along it.

### Mini Comic Panels

Good for: failure to success, the real process, venting/complaints, before-and-after changes from using something.

How to draw it: 2-4 small scenes, each panel expressing only one action.

## Method for Generating Original Metaphors

Reinvent the metaphor from the current article every time — never reuse an old illustration as-is.

### Three Steps

1. Turn the abstract concept into a physical action: getting stuck, leaking, getting heavier, sorting, settling, fermenting, opening a door, folding, unpacking, flowing back.
2. Turn the system structure into a low-tech object: a broken machine, a cardboard box, a drawer, a pipe, a mailbox, a strange gauge, a scale, a well, a ladder, an odd workstation.
3. Have Xiaohei perform the action — not standing off to the side, but stuck inside the machine, pulling the wrong cable, guarding a gate, hauling things, patching something up, weighing something, holding a ladder steady, taking notes, or stuffing something into some strange contraption.

### Pool of Usable Objects

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, tangled cord, sluice gate, turntable, black box, hole puncher, dough press, clothesline, strange workstation.
- Pick only 1-2 when using these — don't pile them all in.

### Pool of Xiaohei Actions

- Pull, haul, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, take apart, mark, recycle.
- The action should serve the core meaning — don't be weird just for the sake of being weird.

## Rule Against Repetition

Don't open or replicate `assets/examples/` by default. Those images are only for style calibration — things like line density, white space, restrained color, and Xiaohei's general vibe.

Unless the user explicitly says "follow this one / replicate this composition / adapt this example," don't directly reuse these old compositions:

- A conveyor belt with two break points
- Xiaohei pulling a judgment lever inside a content machine
- Xiaohei turning into a funnel to sort traffic/trust/conversion
- Xiaohei cutting up a fish made of material/assets
- Xiaohei pulling along a delivery path
- Xiaohei pulling three layers of information sources
- Three Xiaoheis each holding a megaphone/building a bridge/opening a door
- Xiaohei stamping a toolbox of talking points
- Xiaohei holding up a sign showing the path of common pitfalls

Swap in a new metaphor even for the same kind of topic. For example, a "delivery path" doesn't have to be drawn as a route — you could instead draw Xiaohei connecting the tail end of some content to a doorknob; "getting multiple uses out of one thing" doesn't have to be drawn as a fish — you could instead draw Xiaohei pressing a balled-up piece of paper into several different shapes.
